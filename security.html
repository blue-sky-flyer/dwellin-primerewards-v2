Data Security

How we protect your home maintenance records and personal information

Data Security at Dwellin

At Dwellin, protecting your data is our top priority. Your home maintenance records and personal information are among your most valuable assets. This page explains the security measures we implement to keep your data safe.

Encryption

In Transit

All data transmitted between your device and Dwellin servers is encrypted using industry-standard TLS 1.2+ (Transport Layer Security). This ensures that information is protected while traveling across the internet.

At Rest

Your stored data is encrypted using AES-256 encryption (Advanced Encryption Standard). This military-grade encryption means that even if someone gained physical access to our servers, they could not read your data without the encryption keys.

Authentication & Access Control

Secure Login

Your Dwellin account is protected by:

  • Secure password hashing using industry-standard algorithms
  • Optional two-factor authentication (2FA) for additional security
  • Session management to prevent unauthorized access
  • Automatic logoff after periods of inactivity

Employee Access

Dwellin employees only have access to data necessary for their specific job functions. We enforce:

  • Role-based access control (RBAC)
  • Minimum necessary access privileges
  • Regular access audits and reviews
  • Signed confidentiality agreements with all team members

Infrastructure Security

Cloud Hosting

Dwellin uses Amazon Web Services (AWS) for secure, reliable hosting. AWS maintains:

  • Multiple data center redundancy
  • Automatic backups and disaster recovery
  • DDoS protection and mitigation
  • SOC 2 Type II compliance
  • HIPAA eligibility and FedRAMP authorization

Network Security

Our infrastructure includes:

  • Firewalls and intrusion detection systems
  • DDoS mitigation and rate limiting
  • Web application firewalls (WAF)
  • Virtual private network (VPN) access for sensitive operations
  • Regular network penetration testing

Application Security

Secure Development

Our development process prioritizes security:

  • Secure coding practices and training for all developers
  • Code review process for all changes
  • Static and dynamic code analysis
  • Dependency scanning for vulnerable libraries

Testing & Scanning

Before release, Dwellin undergoes:

  • Regular security scanning and vulnerability assessments
  • Annual third-party penetration testing
  • OWASP Top 10 vulnerability testing
  • Compliance scanning and validation

Compliance & Standards

Dwellin adheres to industry-leading security and privacy standards:

  • GDPR: General Data Protection Regulation (EU)
  • CCPA: California Consumer Privacy Act
  • SOC 2: Service Organization Control compliance
  • OWASP: Open Web Application Security Project standards
  • Industry Best Practices: NIST Cybersecurity Framework, CIS Benchmarks

Data Backup & Recovery

Backups

Your data is backed up:

  • Continuously and automatically
  • Multiple times per day
  • In geographically redundant locations
  • Using encrypted backup systems

Disaster Recovery

In the event of a disaster, Dwellin has:

  • Documented disaster recovery procedures
  • Regular recovery testing and validation
  • Commitment to restore service within 24 hours
  • Transparent communication with affected users

Third-Party Security

When Dwellin uses third-party services (payment processors, cloud providers, analytics), we ensure:

  • Signed Data Processing Agreements (DPA)
  • Security audit verification
  • Compliance with GDPR and CCPA requirements
  • Regular vendor security assessments

Incident Response

Detection & Notification

If a security incident occurs, Dwellin will:

  • Immediately investigate and contain the incident
  • Assess the scope and impact
  • Notify affected users as required by law (typically within 30-60 days)
  • Provide clear information about what happened and recommended actions

Remediation

After any incident, we:

  • Implement fixes to prevent recurrence
  • Conduct post-incident reviews
  • Update security procedures and controls
  • Document lessons learned

Your Role in Security

While Dwellin implements comprehensive security measures, you play an important role:

  • Strong Passwords: Use unique, complex passwords
  • Two-Factor Authentication: Enable 2FA on your account
  • Device Security: Keep your phone and computer updated with security patches
  • Public WiFi: Avoid accessing sensitive information on unsecured networks
  • Phishing Awareness: Be cautious of suspicious emails claiming to be from Dwellin
  • Report Issues: Notify us immediately of any security concerns

Security Audits & Updates

Dwellin maintains security through:

  • Annual third-party security audits
  • Regular vulnerability assessments
  • Continuous monitoring and logging
  • Regular security training for team members
  • Quarterly security reviews and updates

Contact Security Team

If you have security concerns or want to report a vulnerability, please contact our security team:

Email: security@dwellin.com
Response Time: We aim to respond to security reports within 24 hours

Responsible Disclosure

We appreciate responsible disclosure of security vulnerabilities. Please email security@dwellin.com rather than publicly disclosing vulnerabilities to allow us to fix issues before public announcement.

Privacy-First Design

Security is built into Dwellin from the ground up:

  • Data Minimization: We only collect data we need
  • User Control: You control what data is shared and with whom
  • Transparency: We're clear about how we use and protect your data
  • Accountability: We take responsibility for your information

Changes to This Security Policy

We may update this Data Security page as our security practices evolve or new threats emerge. Material changes will be announced via email and on this page. Last updated February 2024.
